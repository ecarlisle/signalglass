# Spec 011: CLI

## Status

Implemented

## Purpose

Define the command-line interface for SignalGlass.

## Scope

- Offline analysis command.
- Live ingress command.
- Trace management commands.

## Non-goals

- Interactive TUI.
- Configuration wizard.

## Required files or modules

- `packages/cli/src/cli.ts`
- `packages/cli/package.json`

## Required types or contracts

- CLI positional commands and flags.

## Required behavior

Offline analysis:

```bash
signalglass analyze <file> [--report terminal|json|html] [--output <file>]
```

Live ingress:

```bash
signalglass ingress --config <file> [--port <port>] [--storage <file>]
```

Trace management:

```bash
signalglass traces --storage <file> list [--report terminal|json] [--output <file>]
signalglass traces --storage <file> show <id> [--report terminal|json|html] [--output <file>]
```

- The CLI wires packages together and contains no domain logic.
- Help text is accurate and concise.

## Acceptance criteria

- [x] `signalglass analyze samples/messy-agent-run.json` works.
- [x] `signalglass ingress --config <file>` starts the ingress and accepts optional port and storage flags.
- [x] `signalglass traces --storage <file> list` lists stored traces.
- [x] `signalglass traces --storage <file> show <id>` renders a stored trace.
- [x] Unknown commands print usage and exit with non-zero status.
